Core Primitives

The technical substrate that makes Pharos genuinely different from generic EVM chains.

x402 Payment Rails

Circle CCTP + Chainlink CCIP

Architecture

How Pharos Is Actually Built

A layered parallelism model addressing throughput, storage, and extensibility simultaneously — designed to target DP4/DP5 degree of parallelism.

L1-Core
Decentralized High-Throughput Chain
Subsecond finality via Asynchronous BFT consensus. Parallel transaction execution with full lifecycle async pipelining.
L1-Base
Hardware-Accelerated Data Availability
High-performance data availability layer with Parallel Merklization and real-time verification for seamless TradFi integration.
L1-Extension (SPNs)
Special Processing Networks
Pluggable heterogeneous compute layer. Enables privacy-enhanced computing, native restaking, and cross-SPN interoperability.
PVM + Dual Execution
EVM + WASM Runtime
Pharos Virtual Machine runs both EVM and WASM. Parallel Merklization + Indexed storage enables real-time asset verification.
Consensus
Async BFT — DP4 Target
Five-tier Degree of Parallelism model. DP4 combines scalable consensus, parallel execution, async pipelining, and high-performance authenticated storage.
Cross-Chain
Chainlink CCIP + Circle CCTP
CCIP is the canonical cross-chain messaging layer. CCTP domain 31 handles native USDC burn-and-mint. Chainlink Data Streams supply RWA market feeds.
